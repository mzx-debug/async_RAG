## 核心概念

**(xE, xR)**：embedding/retrieval 设备。0=CPU, 1=GPU。

**成本模型 (v4) — 物理正确模型**：
```
# wall_q = wall_total / B（per-query ms/q）：
#   wall_q = gen_per_token × avg_output_tokens
#            + queue_penalty
#            + gpu_contention          (仅 xE=1)
#            + 0.05 × (emb_q + ret_q)  (xE=0 时，E+R 与 Gen 完全 overlap)
#            + xfer_q                   (仅 xE≠xR)

# Gen：vLLM continuous batching 下为常数（0.378 ms/token，跨 action 稳定）
# Queue：异步调度 per-query 开销（2.5 ms/q）
# GPU Contention：GPU emb 和 Gen 串行化惩罚（38.5 ms/q，仅 xE=1）
# Emb+Ret：(xE=0) 完全 overlap Gen → 仅 5% overhead；(xE=1) 计入 gpu_contention
# Xfer：跨设备传输速率 × L（xE=xR 时 = 0）

# T_cycle（无 overlap 假设）：
#   (0,0): gen + queue + 0.05×(emb+ret)
#   (0,1): gen + queue + 0.05×(emb+ret) + xfer_q
#   (1,0): gen + queue + gpu_contention + 0.05×ret_q
#   (1,1): gen + queue + gpu_contention

# 参数含义（RTX 4090 实测）：
#   gen_per_token    ms/token   生成吞吐率（常数，约 0.378）
#   avg_output_tokens -          平均输出 token 数（EMA 自适应）
#   queue_penalty    ms/q       调度开销（约 2.5）
#   gpu_contention   ms/q       GPU emb+gen 串行惩罚（约 38.5，仅 xE=1）
#   e[xE]            ms/token   embedding 速率（0: CPU=0.084, 1: GPU=0.016）
#   r[xR]            ms         retrieval 系数（0: CPU=0.68, 1: GPU=0.50）
#   alpha[xR]        -          retrieval 亚线性指数（0: 0.55, 1: 0.30）
#   K[xE,xR]         ms/token   传输速率（0→1: 0.55, 1→0: 0.16）
```

## 已知问题 & 解决

### CUDA 不可用
- **现象**：`torch.cuda.is_available()` 返回 False
- **原因**：未安装 CUDA 版 PyTorch，或 conda 环境未激活
- **解决**：`pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124`

### vLLM 未安装
- **现象**：`No module named 'vllm'`
- **解决**：`pip install vllm>=0.6.0`

### FAISS GPU OOM（cublas error 13）
- **现象**：`Assertion 'err == CUBLAS_STATUS_SUCCESS' failed in runMatrixMult`
- **原因**：vLLM 显存占用过高，FAISS GPU 矩阵运算无足够 VRAM
- **解决**：降低 `--gpu-memory-utilization` 至 0.45，或用更小的生成模型

### FlashInfer JIT 失败
- **现象**：`RuntimeError: Ninja build failed...`
- **解决**：`rm -rf ~/.cache/flashinfer/`

### 模型下载慢/失败
- **解决**：`export HF_ENDPOINT=https://hf-mirror.com`

## RTX 4090 Laptop 16GB 实测结果

### 成本模型标定结果（v4）

| Action | 预测 wall (B=64) | 实测 wall | 误差 | 排名 |
|--------|-----------------|-----------|------|------|
| xE0R0 | 3063ms | 3214ms | **-5%** | 1 |
| xE0R1 | 3063ms | 3302ms | **-7%** | 2 |
| xE1R0 | 5527ms | 5218ms | **+6%** | 3 |

**Action 排名完全正确**：xE0R0 ≈ xE0R1 < xE1R0 < xE1R1

详细标定过程见 [docs/cost-model-calibration.md](docs/cost-model-calibration.md)

### Pipeline 性能对比

| 模式 | Wall(s) | QPS | 说明 |
|------|---------|-----|------|
| serial | 30.9 | 8.3 | 基线，固定 batch，串行 |
| async_plain | 30.1 | 8.5 | 预建批次的简单异步 |
| **async_v2** | **23.7** | **10.8** | EMA 在线调度，动态 batch |

加速比：异步 **+30% QPS**，相比基线 **1.3x** 加速。
