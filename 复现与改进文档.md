## 核心概念

**(xE, xR)**：embedding/retrieval 设备。0=CPU, 1=GPU。

**成本模型 (v3)**：
```
# 各分量公式（总时间，ms/batch）：
#   emb:  L_in * e[xE] * B         ← embedding 计算（e：ms/token）
#   ret:  r[xR] * B^{alpha[xR]}    ← 检索（亚线性，r：ms）
#   gen:  P0 + p_lin * L_in * B + g * B   ← 生成（prefill + decode）
#   xfer: I(xE≠xR) * K[xE,xR] * L_in * B  ← 跨设备传输

# wall_q = wall_total / B（per-query ms/q）：
#   emb_q = L_in * e[xE]                  （常数，不含 B）
#   ret_q = r[xR] * B^{alpha-1}          （B 增大时减小）
#   gen_q = P0/B + p_lin*L_in + g        （P0/B 项随 B 增大而减小）
#   xfer_q = I(xE≠xR) * K[xE,xR] * L_in  （常数，仅跨设备时）

# T_cycle 按 (xE, xR) 组合：
#   (0,0): max(gen_total, emb+ret) / B            CPU E+R || GPU gen
#   (1,0): emb + xfer + max(ret, gen) / B        GPU emb → CPU ret || gen
#   (0,1): emb + xfer + max(ret, gen) / B        CPU emb → GPU ret || gen
#   (1,1): (emb + ret + gen_total) / B            全 GPU 串行

# 参数含义与默认值：
#   e[xE]  ms/token  embedding 速率，按 xE
#   r[xR]  ms        检索系数，按 xR
#   alpha[xR]  -      检索亚线性指数，默认 0.5
#   P0     ms         生成 prefill 固定开销
#   p_lin  ms/token   生成 prefill 每 token 系数
#   g      ms/q       生成 decode 每 query 成本
#   K[xE,xR] ms/token 传输速率，xE=xR 时 = 0
#   contention[xE,xR]  obs/pred 比值，吸收模型残差和 GPU 争用
```

## 已知问题 & 解决

### CUDA 不可用
- **现象**：`torch.cuda.is_available()` 返回 False
- **原因**：未安装 CUDA 版 PyTorch，或 conda 环境未激活
- **解决**：`pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124`

### vLLM 未安装
- **现象**：`No module named 'vllm'`
- **解决**：`pip install vllm>=0.6.0`

### FAISS GPU OOM（cublas error 13）
- **现象**：`Assertion 'err == CUBLAS_STATUS_SUCCESS' failed in runMatrixMult`
- **原因**：vLLM 显存占用过高，FAISS GPU 矩阵运算无足够 VRAM
- **解决**：降低 `--gpu-memory-utilization` 至 0.45，或用更小的生成模型

### FlashInfer JIT 失败
- **现象**：`RuntimeError: Ninja build failed...`
- **解决**：`rm -rf ~/.cache/flashinfer/`

### 模型下载慢/失败
- **解决**：`export HF_ENDPOINT=https://hf-mirror.com`
