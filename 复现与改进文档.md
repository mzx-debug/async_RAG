## 核心概念

**(xE, xR)**：embedding/retrieval 设备。0=CPU, 1=GPU。

**成本模型（V4）**：
```
wall_single[(xE,xR)] = max(gen_time, emb_ret_time)    # xE=0, CPU embed
                      = gen_time + er_time × overlap  # xE=1, GPU embed

gen_time = gen_base + gen_per_sqrt × √batch_size  # per-(xE,xR) 独立参数
```

所有参数通过 EMA 从实际执行数据中持续校准。

```bash
# 第一次运行：校准并保存
```

## 已知问题 & 解决

### CUDA 环境问题
- **现象**：`torch.cuda.is_available()` 返回 False，或 `cuInit` 返回 999 (unknown error)
- **原因**：系统启动后 GPU driver 未就绪（重启前遇到，重启后恢复）
- **解决**：重启电脑。使用 `source /home/cloudteam/Software/conda/bin/activate p702`
- **验证**：`python3 -c "import torch; print(torch.cuda.is_available())"` 应输出 `True`

### FlashInfer JIT 缓存问题
- **现象**：vLLM 启动时报 `RuntimeError: Ninja build failed... missing and no known rule to make it`
- **原因**：FlashInfer JIT 缓存指向了错误的路径（旧的 venv 路径）
- **解决**：`rm -rf /home/cloudteam/.cache/flashinfer/`
